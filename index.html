Help Center Glossary

Enrolled / unenrolled (active / inactive): learner account status defined in the profile in the Mission Center. There are two possibilities for a learner account to go into inactive status: the learner has not been connected during a defined period or its status is modified by a manager. When a learner account is inactive, the learner can no longer log in.

App: the App or application is the Teach on Mars tool installed in a mobile device

Learners: people with an account accessing the App or the Webapp allowing them to complete training courses, receive communications, ...

Badges: reward in the form of a badge allocated according to the result of a learner on an activity (success, flawless) or training course (total completion)

Catalog: set of training courses arranged in categories and appearing for a learner in his/her training tab.

Category: grouping of several training courses appearing in the catalog

Custom field: additional field to the default fields for grouping learners and thus use filters to provide access to training courses, send communications and analyze statistics

Certification: reward delivered in the form of a certificate by email to the learner for having successfully completed the scoring activities of a certifying module of a training course

Completion: term used when an activity, module or training course has been fully completed

In training: indicates that the learner has started at least one training course

Training course: it is the course to be carried out, composed of modules, themselves composed of activities

Direct link to a training course or an activity: this link is requested by the Teach on Mars support team in order to be able to analyze and come to assistance more quickly. This is the direct link leading to the training course or to a training activity in the Mission Center

Live!: mobile activity that is played face-to-face and in real time. The Live! is a paid option, which can be activated on demand

Managers: people with an account accessing the Mission Center allowing them to perform several actions (translations, creation of training courses, creation of learner accounts, creation of communications, etc.)

Mission Center: administration platform allowing managers to perform several actions (translations, creation of training courses, creation of learner accounts, creation of communications, etc.)

OTS: or on-the-shelf contents, duplication of training courses from a supplier Mission Center to the client Mission Center

Points: reward delivered in the form of points. All the activities earn points counted in the learner statistics (by activity or by training course), with the exception of Toolbox, Self-evaluation and exercises

Progress: percentage showing the progress in terms of the activities carried out / started / completed by the learner within a training course

Score: learner's result in an assessment activity, reduced to 100%. This score will be compared to the pass threshold set for the activity to determine whether it was successful or not

Success: term used when an activity has been successful based on the success threshold defined in the activity configuration

Test version: status of a training course allowing the Manager to review and validate the training course before publishing it and thus making it accessible to learners registered for the training course. The status of content reviewer is necessary to have access to the training in Test version.

Webapp: if the learner cannot access the Teach on Mars application from a mobile device, he or she will access the application from a computer (desktop) to the Webapp (Web application)

How to correctly display an export file containing special characters?

It may happen that when opening your export file, all the data is in one line, or that special characters are not correctly displayed.

The problem is that the file opens without using UTF-8 encoding, so the content is badly interpreted.

 

How to modify the file?

If you do not find how to set Excel® to force it to open in UTF-8, the easiest way is:

1) Open the csv file with a simple text editor (Notepad, TextEdit, Text Wrangler, Sublime Text ...), you can now check if the characters are correctly displayed

2) Open a new Excel® document
3) Copy/Paste all the csv file content opened in your text editor in the first cell (A1) of the Excel® document.

 

How to convert my file?

1) Select the first column A (in which is all the content that has been pasted)
2) Click Data / Convert
3) A popup opens, click Next then Choose the separator "semicolon" in the 2nd window, then End
4) Everything is correctly tabulated, and the good characters have been preserved.

How are the statistics handled if the learner's device changes?

The Teach on Mars app can be installed on several devices, with a same learner account.

 

What happens if the learner changes devices?

At each connection only and on each device, synchronize automatically:

  • progress
  • assiduity
  • time spent
  • badges

The score, level and duels are also synchronized in real time, as soon as the device is online.

In the case of public access (without authentication), each connection to a new device creates a new account (via the creation of a new nickname), so there is no statistics synchronization process.

How to use your multilingual App?

An option always allows the learners to change the language, for the interface and for each training course (during its download, but also after).

The application detects the language of the learner with the following rules: 

1) The application analyses the device language. It compares the device language to the ToM Enterprise interface available languages. A default language is always defined. There are 2 options:

  • The device language is defined in the application available languages. In that case, the application selects the device language for its interface.
  • The device language is not defined in the application available languages. In that case, the behavior depends on the application format: the application selects its defined interface default language, for its interface.

2) The training language may differ from the interface language. When learners download a new training, the application checks the active interface language. There are 2 options:

  • The active interface language is defined in the training available languages. In that case, the interface language is proposed
  • The active interface language is not defined in the training available languages. In that case, the training default language is proposed.

    

Note: 

1. The learner can choose his/her default language (interface language) from the login page

2. The learner can modify his/her language from the profile tab

 

What happens to languages that read from right to left (RTL or "Right-to-left" languages)?

As from version 18.1, the Teach on Mars solution manages training course content in languages that read from right to left (such as Arabic or Hebrew).

Note: This applies to content only. The app's interface remains in its original language (languages read from left to right only). The RTL language will not work if it is preceded by a latin character.

 

 

Can you access a training offline?

 

The learner can access an offline training course, since he has downloaded the training course beforehand, it is accessible everywhere.

Training courses are downloaded in local mode except for videos, which are by default viewable in streaming (for more information, read the article "How to manage videos in training course activities?").

 

What is the difference between the online and the offline mode?

When downloading a training course, the learner chooses either the online mode (default option) or the offline mode. This option only applies to videos.

  • If the training is downloaded in online mode, it will be accessible offline except for the videos and training courses updates.
  • If the training is downloaded in offline mode, it will be accessible offline with videos except for training courses updates.

Once all the cards have been swiped to the right (the learner thus indicating that all the knowledge in the activity has been assimilated), the progress bar reaches 100% and the activity restarts to allow the learner to revise again. What is the Recommendation Quiz?

SO WHAT EXACTLY IS THE RECOMMENDATION QUIZ?

The Recommendation Quiz is an activity designed to enable you to offer your learners a personalized learning path, based on their score in a questionnaire-style quiz. 

Every question in the quiz relates to a specific theme, which in turn corresponds to one or several training courses in your catalogue.

When your learners do not get enough correct answers on a given theme you can recommend training courses that will help them reinforce their knowledge and encourage them to start improving in those areas where they are not comfortable. 

You can also recommend training courses to learners on themes where they have scored highly. That way they can further develop their expertise on those topics which clearly interest them the most.

In addition, you can enable an Global Evaluation option, which will give your learners insight into their overall level of knowledge on all the themes covered in your Recommendation Quiz.

 

HOW DOES THE RECOMMENDATION QUIZ WORK?

The Recommendation Quiz is a training course that only includes one activity. It is therefore ideal for promotion directly from the Wall, encouraging your learners to download it straight away at their very first connection to your app. Once they have completed it, they will be guided towards their recommended learning path based on the answers they give and the score they achieve.

  

DESIGNING YOUR RECOMMENDATION QUIZ

Before you start creating your Recommendation Quiz in the Mission Center, you will need to be clear on what your objectives are: 

- What are the main themes featured in your training catalogue? Are you looking to cover all these themes in your Recommendation Quiz? 

- What is the target population for your Recommendation Quiz? Do all the learners you are targeting have access to the training courses you'd like to recommend? 

- Are you intending to use your Quiz to recommend "expert" training courses to those learners who score highly? Is the plan to recommend training courses that will fill any knowledge gaps or shortfalls on specific themes that your learners' responses identify? Or are you looking for your Quiz to generate recommendations in both these scenarios? 

- As well as evaluating the knowledge of your learners on each theme, do you want to add an global evaluation across all your themes? 

- How are you going to communicate on and distribute your Recommendation Quiz? If you want every new learner to complete the Quiz as soon as they access your app for the first time, have you thought about creating a dedicated Category at the beginning of your training catalogue?

 

SOME BEST PRACTICES

Structure your themes

Start by defining the themes for your Recommendation Quiz. You can incorporate up to 10 different themes – but remember that each one will need to have recommended training courses associated with it! You can start from your training catalogue and extract the themes that you build your Recommendation Quiz around. Or conversely, you can begin by defining the themes you want to evaluate, and then identify two or three training courses from your catalogue that fit with them. A recommended training course can be anything from a single Quiz Game right though to a much more sophisticated course with multiple modules and activities.

Write your questions

Every question in your Recommendation Quiz must be associated with one of your themes, and each question has 2 or 3 possible answers.
Each answer corresponds to one of three "levels": correct, incorrect or neutral. 

Every theme must have at least one question associated with it. In fact, we recommend a minimum of 3 questions per theme to ensure you get the most meaningful and accurate results possible. 

Think about your options

There are two optional parameters you can set when you create your Recommendation Quiz.

Firstly, you can opt for an global evaluation of your learner in addition to an evaluation for each theme: the global evaluation aggregates the results of the learner for all the questions you ask in your Quiz, across all your themes. 

You can also activate the option that enables the learner to complete the Recommendation Quiz several times: every time he completes the Quiz, the learner can receive new training course recommendations according to his responses to the questions. 

Communicate on the Wall

Don't forget to use the Wall to let your learners know about the Recommendation Quiz when you publish it! If you want all your new learners to complete the Recommendation Quiz the first time they connect to your app, you will have to publish a "Training course" or "Article" communication with a direct link to your Recommendation Quiz in first position in your Wall Slider. 

You can also create a specific Category for your Recommendation Quiz that you can then position at the very beginning of your training catalogue. 

OK, so now you're clear on the objectives and the design of your Recommendation Quiz, let's get down to business and build it! 

To do this, see our article "How to create a Recommendation Quiz?"

 

 

How to limit the access of a module over time?

Teach on Mars allows you to lock a module according to different criteria:

  • By code (read the article « How to unlock a module using a code ? »)
  • Under condition of success of another module 
  • By date: this is what we will see in this article!

This feature allows you to pilot a distance learning program over time. You can thus ensure the provision of training course modules one after the other, several days or several weeks apart.

By locking the modules over time, you can also make one or more modules of a course accessible over a given period. This allows you to set up training course campaigns.

 

> CATALOG / Training courses > your training course > Administration

In the Modules access management window, click on Create . 

 

1. Enter dates of availability

Select the module for which you want to indicate locking dates. Choose your start and end dates.

You can choose to indicate only a provision date in the “From” cell and leave the “To” cell empty: learners will have access to the module at from the date entered, with no time limit thereafter.

But you can also choose to indicate only a date in the “To” cell: in this case, learners will have access to the module as soon as they are published (provided they have access to training course!), but for a limited time, until the date indicated.

 

2. Filter access by population

You can choose to fill in the filters from the custom fields, in order to propose locking conditions for one or more training modules for some of the learners who have access to the training course. You must create a locking condition rule by population (it is not possible to create a rule for several groups at the same time). If the rules concern all learners who have access to the training course, then it is not necessary to fill in the filters.

 

Note: the locking conditions are automatically disabled if a Profiling activity is present in the training path and recommends activities. If the Profiling does not recommend any other specific module, the condition will work.

How are learners points calculated?

Learners earn points every time they perform an activity. If ranking is activated in a specific training course, learners are ranked according to their number of points in this specific training course. Information about learners' points is also available in the Mission Center.

Quick reminder: points are different from the score (see the article about score :"What is a scored activity?")

Learners can earn points with all types of activity. The amount of points earned depends on performance and number of attempts. It follows different rules depending on the type of activity.

Here's how points work for each family of activities:

 

1/ Non-scored activities (except Flash Game and Brainstorming)

Non-scored activities are: Mobile Course, Learning Quiz, Flash Course, Mixed Course, Survey, Profiling

  • When the learner ends such an activity for the first time, 10 points are earned.
  • When the learner ends the activity again later, 1 point is earned.
  • Once the learner has finished this activity 5 times, no more points will be earned.

 

2/ Flash Game

  • When a learner sees a card for the first time, 2 points are earned
  • When a learner returns a card, 2 points are earned
  • When a learner sees a card they have already seen, 1 point is earned
  • Once a learner has finished this activity 5 times, no more points will be earned.

 

3/ Scored activities (except Quiz Game)

Scored activities are: Challenge, Fill the Gap, Pick a Word, Training Game, Wordspool, Sushi Game

  • The amount of earned points is based on a "Maximum" of 50 points. When the learner succeeds at a scored activity, they earn a percentage of that "Maximum", which correspond to their percentage of correct answers. E.g. With 80% of correct answers, the learner earns 40 points. (80% x 50 points = 40 points).
  • Bonus: If the learner answers all questions with no mistake, the amount of points is doubled
  • Bonus: If the learner succeeds at the activity on the first attempt, the amount of points earned is doubled
  • Once the learner has finished this activity 5 times, no more points will be earned.

Example: if a learner succeeds at a scored activity with no mistake on first attempt, he earns 200 points (=50x2x2).

 

4/ Quiz Game

Learners earn points at the end of a Quiz Game according to these rules:

  • A correct answer to a newly viewed question is 5 points
  • A correct answer to a question they have already replied to is 1 point
  • If the learner reaches the target before the end of the timer, they earn a time bonus. For example, if there is 20% of the whole time remaining, they earn a 20% bonus on the points earned from the correct answers
  • If the learner gives only correct answers, points are doubled.

Example: If a learner scores on first attempt 10 correct answers out of 10 in 40 seconds (the whole timer being 50 seconds), the amount of points is: 

  • 10 answers x 5 points = 50 points
  • which are doubled since the learner answered all questions correctly
  • with a time bonus of (20% x 50 points =) 10 points

The final result is 110 points.


There are also bonuses related to Quiz Game in duel mode, which are added after the opponent answers the duel.

  • 50 points when the learner wins
  • 10 points when the learner loses
  • 25 points in case of a tie between players


NB: Quiz Game is not affected by the limit of points after a number of attempts.

 

5/ Brainstorming

Learners earn 1 point for each vote on a proposal they have posted in a Brainstorming activity.

 

6/ Recommendation Quiz

Even if the Recommendation quiz is not an activity but a training course type, learners can also earn points:

  • When the learner ends such an activity for the first time, 10 points are earned.
  • When the learner ends the activity again later, 1 point is earned.
  • Once the learner has finished this activity 5 times, no more points will be earned.

 

Note: Toolbox and Self-Evaluation/Exercises are not concerned by points.

What are used multi-values fields for ?

These are used in the profile, for example, to perform a search. However, they cannot be used subsequently for the filters of the access rules or the rules for sending communication. Values ​​must be separated by commas ",".

Example:

  • A learner works for two subsidiaries of a client. There is a Subsidiary custom field.
  • Several subsidiaries companies exist for a client (Lyon, Paris, Marseille, Bordeaux).
  • The learner working for Paris and Marseille will have in his profile under the Subsidiary field: Paris, Marseille
  • When we do our search to see all the learners in "Paris", this one will appear together will all learners in Paris. It will be the same if we carry out a search containing "Marseille" together will all learners in Marseille. We will also find by searching for "Paris, Marseille" but we will not find it by searching for "Marseille, Paris".

Therefore, it can be concluded that in the profile, the comma is considered as and and, but in a search, the comma takes the full string of characters.

You can put two values in a field on a profile. If we do a search, on one or the other value, the learner will appear in both cases. You cannot put two values in a field when you do a search, or when you select a population of recipients.

How does moderation work for comments on the Wall?

Comments published on the Wall are moderated in two stages:

  1. The comment may be reported by a learner in the app
  2. The manager then accepts or rejects the comment via the Mission Center 

 

FROM THE APP TO THE MISSION CENTER

Learners can report a Wall comment that they consider offensive or inappropriate. To do so, they simply press and stay pressed on the comment in question and select Report. 

Reported comments are forwarded to the Mission Center in the Communications > Comments moderation section. 

> COMMUNITY / Communications  > Comments moderation

For each reported comment, the manager can open the communication (in the web application) and view the associated comments. The manager then chooses whether to accept or reject the reported comment. 

You can filter the reported comments according to their status: 

  • Not yet moderated (comments newly reported) 
  • Moderated (log of moderated comments) 
  • All reported comments 

The comment has been accepted? It remains visible on the Wall. 

The comment has been rejected? It disappears from the Wall. Instead, a comment This comment has been moderated appears.

 

PERMISSIONS TO REPORT AND MODERATE 

Within the application, all learners who can comment on a communication are also entitled to report a comment on the same communication.

What is the On-the-Shelf feature?

The On-the-Shelf feature allows owners of a source Mission Center to deliver content to other Mission Centers without sharing the data / design files:

  • either fully (by generating user licenses), or partially unlocked (by sharing a demo),
  • to monitor the use of shared content, by both the supplier and the customer,
  • manage updates to shared content.

You can use the "On-the-Shelf" feature:

  • If the source and destination Missions Centers are in version 18.1 or later (to be checked at the bottom of the Mission Center after the "Powered by Teach on Mars" mention).

  • If the option is activated on the source and destination Missions Centers, bringing up the "Generated licenses" / "Acquired licenses" menus when you click on Training courses).

If not, contact your project manager.

To share a content, the actions to perform are as follows:

  1. the client requests the content from the destination Mission Center (only for their first request), which notifies the supplier by email,
  2. the supplier generates the license from the administration page of the requested training course, which notifies the customer by email,
  3. the client imports the training course, to the destination Mission Center.

The client can then manage the training course received on the destination Mission Center while the supplier can continue to manage the editing functions on the source Mission Center.

How to use the Mission Center Web services

Request and response structure

The API is HTTP and JSON based. Data passed in POST and PUT requests is stored in the body as JSON. In responses, the data is also stored in the body response and structured as JSON. 

To be able to send requests to these webservices, you need an API key and an API secret, but you also need to construct your request in a certain way so that it is accepted by the server. API credentials and context information are stored in the HTTP headers. The authorization protocol is designed to include elements that change from one request to another, so the headers will have to be constructed dynamically. We will see how in the next section.

 

Teach on Mars API Headers reference

In order to call any ToM service, some HTTP headers must be sent to authorize the request. Here's a list of the headers required in any request send to the Mission Center.

X-TOM-API-KEY The API key that you've been given
X-TOM-APP A reference code that represents the application
X-TOM-RTS The Unix timestamp of the request (seconds since Jan, 1 1970)
X-TOM-NONCE A random string that must be different between each request
X-TOM-API-HASH The hash string that is used to validate the request

 

API hash construction

SECRET = The API secret linked to this API key
RTS = X-TOM-RTS Header
APP = X-TOM-APP Header
REQUEST_HASH = MD5 ( BODY REQUEST_URL)
NONCE = X-TOM-NONCE Header

STR_TO_ENCRYPT = CONCAT( SECRET RTS APP REQUEST_HASH NONCE )

HASH = SHA256 ( STR_TO_ENCRYPT )

 

Code example

Using PHP

$requestTimestamp = time();
$nonce = time().generateRandomString(20);
$hash = $API_SECRET.$requestTimestamp .$APP.md5($data.$wsUrl).$nonce;
$hash = hash('sha256', $hash);

$ch = curl_init($wsUrl);
curl_setopt($ch, CURLOPT_CUSTOMREQUEST, $method);
curl_setopt($ch, CURLOPT_POSTFIELDS, $data);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, array(
 'Content-Type: application/json',
 "X-TOM-APP: $APP",
 "X-TOM-API-HASH: $hash",
 "X-TOM-API-KEY: $API_KEY",
 "X-TOM-RTS: $requestTimestamp",
 "X-TOM-NONCE: $nonce",
));
$result = json_decode(curl_exec($ch));

How to identify a learner in my service

It is possible to create a secondary service that brings new features to the app by using data from the Mission Center webservices. This service must be a web based and online application and could be, for instance, a new profiling quiz, a gift shop, a messaging service, etc.

The key part to implementing this kind of service is identifying the learner. This is where the Identity token comes into play. We are going to go over how it should be used step by step.

Workflow

 

1. Learner launches the service

From the Add-on tab, a link to the service is available to the learner. This URL contains placeholders to transfer the required information to the service.

https://my-service.com/launch?learnerId=$(LEARNER_ID)&token=$(IDENTITY_TOKEN)

The learner taps on the add-on icon and the service URL is opened in a web view.

 

2. App requests Identity Token to Mission Center

Because the URL contains $(IDENTITY_TOKEN) the app will send a request to the Mission Center to obtain an identity token for the current logged in learner.

To configure this add-on link, please contact your Teach on Mars Project Manager.

 

3. Mission Center provides Identity Token

The Mission Center gives the token to the app. The $(IDENTITY_TOKEN) placeholder can now be replaced with the identity token.

 

4. App opens service with Identity Token and Learner ID

When the learner opens the add-on, the placeholders are replaced. $(LEARNER_ID) is replaced with the current learner's unique identifier. $(IDENTITY_TOKEN) is replaced with the token that was just obtained from the Mission Center.

Therefore, the service receives:

  • The ID of the current logged in learner
  • An Identity Token linked to this learner

 

5. Service checks Identity Token with Mission Center

When the service is opened by the learner, it needs to make sure the user is the right one and not someone who just opened the service URL in a random browser.

The service can make sure of that with a request to the MC.

Request (see documentation)

POST /api/v3/verifyIdentityToken

{
  "learnerId": "46e11440-ea40-11e7-a88a-3d64957b0864",
  "token": "oM50g1l1ZctOdjHOTRrK"
}

 

6. Mission Center validates Identity Token

Response(see documentation)

The Mission Center will either respond with:

  • Code 200: which means the token is valid
  • Code 404: the token was not found and therefore is not valid

NB: The token remains valid for 2 minutes after its generation by the Mission Center (step 3).

 

7. Service shows interface to the learner

At this point, the service is sure that the learner that is using it is legitimate.

As the service is opened in a web view, it should give a visual feedback or interface to the learner once it has processed everything it needs to process.

Log the learner in the application

Once the learner is created in the Mission Center, you need to obtain an authentication token from the Mission Center and pass it to the application. This will lead to the learner being authenticated in the app. Here's how it works.

Obtain an authentication token

The system has to call a webservice on the Mission Center with the learnerId to obtain an Authentication Token, which is a random and unique string.

The WS to call is (see documentation):

POST /api/v3/device/auth/token
{
  "learnerId": "46e11440-ea40-11e7-a88a-3d64957b0864"
}

The response should be either a 404 code if the learnerId has not been found or something like that:

{
  "response": "67ad8ef9452d5e7cda467238de7865facd765"
}

Which is the authentication token to be used in the next step.

Log the learner in the application

In order to log the learner in the application, the authentication token must be passed to the application by using a special function. Include this piece of code in your page:

function postAuthToken(authToken, isCreation = null) {
  post(`com.teachonmars://globals.login?authToken=${authToken}&profileCreation=${isCreation}`);
}
function post(url) {
  if (window.top == window) {
      window.location.href = url;
  } else {
      window.top.postMessage(url, "*");
  }
}

What you need to do now is call the postAuthToken function with the token, and the application will automatically log the learner in. For example:

<script type="javascript">
 var token = "67ad8ef9452d5e7cda467238de7865facd765";
 postAuthToken(token, true);
</script>

Completion criteria

This parameter defines when the content will notify the Mission Center that the learner has completed the content. There are two options:

User access

In this mode, the content notifies the Mission Center that the learner has completed the content as soon as the content is opened. This means that you will not be able to differentiate learners who have just opened the content and learners who have actually completed it.

Slide views and / or quiz

In this mode, the content will notify the Mission Center that the learner has completed the content on specific conditions:

  • Slide views: the content will be considered completed when the learner reaches a percentage of slides (or a specific number of slides).
  • Quiz: the content will be considered completed when the learner meets the chosen condition among those:
    • Quiz is attempted
    • Quiz is passed
    • Quiz is passed or the quiz attempt limit is reached
  • Both: the content will be considered completed when the learner reaches both conditions (on slides and quiz)

 NB: This content completion information will impact the progress data in the Mission Center data export.

Success criteria

This parameter defines when/if the content will notify the Mission Center of the learner's success (or failure) on the content.

User access

In this mode, the content will notify the Mission Center that the learner has passed the content as soon as the learner opens the content.

Slide views and / or quiz

In this mode, the content will notify the Mission Center that the learner has passed the content on specific conditions:

  • Slide views: the content will be considered passed when the learner reaches a percentage of slides (or a specific number of slides).
  • Quiz is passed: the content will notify the Mission Center of the success when the learner passes the quiz
  • Both: The content will notify the Mission Center of the success when the learner meets those two conditions.

Technical details

SCORM API discovery

Here's where the SCORM should expect to find the SCORM 1.2 and 2004 API objects. 

Platform SCORM 1.2 SCORM 2004
Webapp window.parent.API window.parent.API_1484_11
iOS window.API window.API_1484_11
Android window.API window.API_1484_11

So, depending on the platform the content is played on, the API object will be accessible at a different place. You need to be aware of that when using an API discovery algorithm. For instance, some API discovery algorithms don't look for the API directly in the window object, and this may cause compatibility issues on the mobile platforms.

Commit behavior

The Commit() and LMSCommit() functions are not implemented in the Teach on Mars platform. To make sure that the data is recorded and sent to the Mission Center, a Terminate() (or LMSFinish()) is required.

Some content only make use of Commit and imply that the data will be saved without using Terminate. This is not currently the case in the Teach on Mars platform.

How to integrate a Storyline content

Articulate 360 Storyline can produce SCORM contents and we are going to see how you can integrate them in the Teach on Mars application.

Content layout

Storyline content is usually not optimised for mobile applications, but there a few things that can be done or checked to make sure the user experience on mobile is good.

Responsive or not?

Storyline does not generate responsive contents. Instead, it will adapt the size of the content to the screen without changing the ratio of the content player. This behaviour implies a couple of consequences that you need to be aware of:

  • Most contents are designed in a landscape oriented player, but in any case the content will be adapted to fit inside the screen with black borders to fill the rest of it.
  • Everything is shrunken down, including interface controls like buttons and menus which can be difficult to handle on a mobile phone.
  • Text also gets smaller on a mobile phone screen, and can sometimes be too small for comfortable reading.

What to do about it

Test the content

The first important thing is to test on a mobile phone screen to make sure the content is usable and readable on a small screen. You can also use the preview feature in Storyline which allows you to preview the content on multiple screen sizes. But keep in mind that nothing can replace a human test on a true mobile phone screen to check if the interface is working and the text is not too small.

Force orientation

If your content renders well on a mobile phone only when the screen is in landscape orientation, then you can configure the Storyline player to force the orientation to landscape on mobile phones, tablets, or both.

  1. Open the Responsive playback menu on the right
  2. Select Landscape only for Phones and/or Tablets.

That's it! You can now export your content again.

Navigation

By default, Storyline does not include an Exit button in the content which is essential in Teach on Mars. So you need to create a trigger or a button that will close the content.

Example

If you have no idea how to do such a thing, here is a way to add a menu in the player with this exit action:

  1. Open the Player properties menu
  2. Add a Player tab 
  3. Name it Exit
  4. Choose the alignment Topbar right or Topbar left
  5. Select the action Exit course
  6. and click OK.

 

This will add a menu with an action that allows the user to leave the content. There is one drawback though. If the content didn't already include a menu, one will be added and take some space on the screen, leaving less space for the player itself.

How to integrate a Kumullus content

If you are considering integrating some Kumullus content in your Teach on Mars app, here are a few key things to know and to do.

Content design

Multi-language content

In Teach on Mars activities, one web content is the same for every language in the training course. Which means, one content must carry all translations and must either detect the learner's language or provide a way to choose a language.

Kumullus does not support multi-language content except for video subtitles that can be translated.

But don't give up yet. If you wish to have your Kumullus content translated in your Teach on Mars app, there is a way around it. You can create a package containing multiple contents (one for each language) and automatically display the right translation when the learner launches the content.

It is a bit technical, but the whole process is detailed in our Github project: Multilingual web content in Teach on Mars.

NB: Kumullus content is mainly video, so the package weight will rise quickly if you add up several translations.

 

Content weight

In the Teach on Mars application, training courses are downloaded on the device so that learners can launch them offline. On mobile devices, the internet connection can be unstable and we don't want our users to have a hard time downloading content.

Therefore the weight of training courses (which includes any web activity inside the training course) must not exceed 200 Mo. So, be careful with media assets, especially videos!

More information in the Export setting section.

 

Theme customization

Kumullus allows you to customize a few things in the rendering of the content.

  • The main color
  • A secondary color
  • The logo in the introduction screen
  • The logo in the conclusion screen

To do so, just go to the Preferences menu. Then choose your colors and logo.

Tips:

  • Color: you can use the same colors as your application, but swap primary and secondary colors.
    • Kumullus primary color: Teach on Mars secondary color (or gray)
    • Kumullus secondary color: Teach on Mars primary color
  • Logo: use a PNG image with transparency to ensure a good quality

Android compatibility 

The Android webview has some issues with the navigation from one point of the video to another. In Kumullus contents, it implies that option Go to the end of the hotspot makes the user experience complicated.

So if your audience uses Android phones, we recommend to disable this option on your hotspots. Just leave the option unchecked like this: 

Don't worry, the content works just fine without this option.

 

Export settings

Once your content is ready, reviewed and polished, you will need to export it as a SCORM package. Luckily, this is very simple with Kumullus.

First, let's open the Export menu.

Choose the export type

Select the following settings:

  • Package SCORM
  • SCORM Version: 2004

Select the video quality

Alright, we want the best video quality for learners, but remember that they will download this whole content and that a training course weight is limited to 200Mo.

You can choose the following settings:

  • Available qualities: only 720p

How to integrate Wooclap sessions

Wooclap is a great tool to create and manage real-time interactions with your audience like questions, polls, rating, brainstorming, word cloud, and so on.

When you create an event in Wooclap, you will be given a link that will lead your users to the right session.


The link is available in the top right hand corner of the event screen

There are various possible integrations in Teach on Mars, but they all rely on the usage of this link at the right place. Let's get to it.

 

Integration through the Wall

This is the best choice if you want to use wooclap in the context of an event that is not entirely linked to a training course. There are two ways of doing that:

  • by creating a web link communication, which leads directly to the session
  • by creating an article containing the link, which allows you to create more context before the user joins the session

 

In a web link communication As a link in an article

In the Mission Center:

  • Create a Web link communication
  • Choose an image
  • Type in a title
  • Add the Wooclap link in the link field

In the Mission Center:

  • Create an Article
  • Fill in your content
  • Add a link block in the content
  • Fill in the label of the block
  • Use the Wooclap link as the URL in the block

Once the learner has opened the link:

If it's the first time opening Wooclap, the learner will be asked for a username
 
After clicking Let's play the learner reaches the session and can participate

 

 

Integration in a Training course

The Wooclap link can be put in a training course as a web link activity. This forces the learner to download a whole training course before being able to join the Wooclap session. 

This option is useful if your Wooclap session is related to a training course that needs to be completed before the session. The Wooclap web link activity becomes another step in the training course.

 

Both methods can take advantage of the use of deep links and QRCode to immediately access the session!
